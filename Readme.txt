Husk Render Launcher
Houdini Solaris (husk) でのレンダリングを効率化するための軽量なバッチツールです。GUIによる直感的な設定と、詳細なログ収集、スマートな進捗監視機能を備えています。

フォルダ構造

RenderTool/
├── render.bat            # 起動用メインバッチ
├── scripts/              # プログラム・スクリプト
│   ├── husk_gui.ps1      # 設定用GUI
│   └── husk_logger.ps1   # ログ管理・実行エンジン
├── config/               # 設定ファイル保存先
│   └── settings.ini      # 自動生成される設定データ
└── log/                  # レンダリングログ保存先
    └── render_YYYYMMDD_HHMM.log

主な機能：

・GUIベースの設定: Houdini Bin、USDパス、出力先、フレーム範囲をマウス操作とテキスト入力で完結。
・環境の自動セットアップ: settings.ini に指定した Houdini Bin パスを自動的に環境変数へ通すため、通常のコマンドプロンプトから実行可能。
・スマート進捗監視: 画面を汚さず、1行の更新でレンダリングの進捗（%）と残り時間をリアルタイム表示。
・詳細ログ記録: log/ フォルダ内に実行ごとの詳細なログ（Verbose 3）を保存。
・単一フレームモード: チェックを入れるだけで開始フレームと終了フレームを同期。
・ドラッグ＆ドロップ対応: USDファイルを render.bat にドロップするだけでパスを入力済み状態で起動。
・終了後レポート: 実際に保存された画像パスと、対応するログファイルの場所を最後に明示。
・自動再起動オプション: 長時間の夜間レンダリング終了後にPCを再起動可能。

使い方：

1.起動: render.bat を実行するか、USDファイルを render.bat にドラッグ＆ドロップします。

2.設定:
  ・初回のみ Houdini Bin フォルダ（husk.exe がある場所）を指定してください。
  ・各パスやフレーム範囲を入力します。

3.開始: 「保存して開始」ボタンを押すとレンダリングが始まります。

4.監視: コンソールに進捗が表示されます。エラーが発生した場合は黄色い文字で強調表示されます。

5.完了: 
  簡易レポートが表示されます。
  ・Oキー: 保存先のフォルダをエクスプローラーで表示します。
  ・Lキー: 詳細ログをメモ帳で開きます。
  ・Xキー: ツールを終了しウィンドウを閉じます。

Discord通知の設定方法：

1. 通知を受け取りたいDiscordチャンネルの設定（歯車アイコン）を開きます。
2. 「連携サービス」 > 「ウェブフック」を選択します。
3. 「新しいウェブフック」を作成し、「ウェブフックURLをコピー」をクリックします。
4. 本ツールのGUI右側にある「Discord Webhook URL」欄にURLを貼り付け、Notificationを「Discord」に切り替えてください。
   ※レンダリング失敗時（エラー終了時）には、赤い埋め込みメッセージで即座に通知されます。

技術仕様
Engine: Karma XPU / CPU (husk)
License Mode: Apprenticeライセンススキップ
Output: $F4 シーケンス形式をサポート
Logging: PowerShell による標準出力・標準エラー出力のリアルタイムストリーム処理
